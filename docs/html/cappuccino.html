cappuccino.model module¶

class cappuccino.model.Model(fp_run='', cut=0.0, temp=0.0, nhyperparams=7, CARAMEL_params_list=[], forward_extrap=0.0, backward_extrap=0.0, redshift=0.0, runname=None, agnname=None, posterior_sample=None, fp_posterior='', load_posterior=False, level_cut=[0, 0], **kwargs)¶

This is the primary cappuccino class that will contain all of the information and results for a CARAMEL run.

Parameters:
  • fp_run (str) – Path to the directory where the CARAMEL run results are stored.
  • cut (float) – Fraction of samples to cut from beginning of sample.txt, defaults to 0.0.
  • temp (int or float) – Statistical temperature for postprocessing, defaults to 1
  • nhyperparams (int) – Number of hyperparameters in the CARAMEL continuum model. This must be set properly for cappuccino to work. Defaults to the CARAMEL master branch.
  • CARAMEL_params_list (dictionary) – List of parameters in the CARAMEL sample.txt output file. This must be set properly for cappuccino to work. Defaults to the CARAMEL master branch.
  • forward_extrap (float) – Fraction of lightcurve extrapolated into the future. Must match the value in Constants.cpp. Defaults to 0.0
  • backward_extrap (dictionary) – Fraction of lightcurve extrapolated into the past. Must match the value in Constants.cpp. Defaults to 0.0
  • redshift (float) – Redshift of the AGN. Necessary for some parameter measurements. Defaults to 0.0.
  • runname (str) – Name identifier for the CARAMEL run. Defaults to None.
  • agnname (str) – Name identifier for the AGN. Defaults to None.
  • posterior_sample (numpy array) – Posterior sample array. If included, will automatically create the posterior property.
  • fp_posterior (str) – Posterior sample filepath. If included, will automatically load and create the posterior property.
  • load_posterior (bool) – If posterior_sample and fp_posterior weren’t provided, will attempt to find the posterior_sample file based on the standard file structure and load. Defaults to True.
  • level_cut (list) – If you’d like to specify exact levels rather than using the standard DNest postproccessing scripts. First value is the minimum level, second value is the maximum level.
Returns:

None
